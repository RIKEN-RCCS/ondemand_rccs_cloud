6.2. FX700 Cross-Compilation Environment

The cross-compilation environment is provided as a Singularity container image. Acquire an x86_64 compute node via Slurm and compile using the container image (the example below uses the r340 partition). 

$ srun -p r340 --pty bash
$ singularity shell /cloud_opt/singularity/x86_64/fx700xos_v1.0.30.sif
$ FCCpx test.c -o test.x

6.3. Scratch Area

A temporary scratch area on compute nodes is available. The scratch directory is dynamically allocated by the scheduler, and its path is stored in the environment variable USER_SCRATCH_DIR. This directory is deleted when the job finishes.

6.4. S3 File Access

A service is available to access R-CCS Cloud files via the AWS S3 protocol. Because this enables external access, it is disabled by default. Administrator setup is required to use it. If you wish to use this service, please apply via this link.

Follow the steps below to enable S3 access. These settings can be configured at https://lens3.cloud.r-ccs.riken.jp/lens3.sts/.

  1. Register the directory where the bucket will be created.
  2. Create the bucket.
  3. Generate access keys.

This service is powered by Lenticularis-S3. For details, see https://github.com/riken-rccs/lens3/.

6.5. Notes

  • Do not run jobs on the login node. For long compilations, acquire a compute node and run them there.
