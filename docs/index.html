1. Policies

1.1. Usage Policy

1.1.1. Use of Compute Nodes

  • As this system is a shared resource, users must not occupy computational resources for extended periods through continuous job resubmission or similar practices.
  • Users must not reserve or use computational resources beyond what is necessary.
  • This system is intended for small-scale performance evaluation and research and development purposes; therefore, the following uses are prohibited:
    • Production use (e.g., large-scale training of LLMs, large-scale simulations for real-world applications)
    • Use for providing continuous services
    • Personal use

1.1.2. Use of Login Nodes

Users must not run long-duration or high-load processes on login nodes. Specifically, the following types of processes must be executed on compute nodes:

  • Continuous execution of AI-assisted tools (e.g., Claude Code)
  • Long-running server programs such as JupyterLab or VSCode Server
  • Long-duration compilation processes using a large number of CPU cores

1.2. Operational Policy

1.2.1. Resource Occupancy Control

  • The maximum reservation time for computational resources per job is 24 hours.
  • The maximum amount of computational resources a user may submit (number of nodes × time) is limited to (number of nodes in the target partition × 72 hours).
  • If excessive job resubmission or resource occupation is observed, job control measures will be applied.
  • A scheduling policy will be applied such that users with lower cumulative usage are given higher priority.

1.2.2. Control of Usage Purpose

  • If a user is determined to be in violation of the usage policy, their account may be suspended.
  • If high-load or long-running processes are detected on login nodes, those processes will be terminated.

1.2.3. Disclaimer and Operational Discretion

  • The operations team assumes no responsibility for any loss resulting from the use of this system.
  • No data backup is provided on this system.
  • User support, including responses to system failures, is provided on a best-effort basis.
  • The operations team reserves the right to modify or suspend the system, or restrict access for specific users, without prior notice.
